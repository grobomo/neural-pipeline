# Plan Phase Reference

## Purpose
Determine HOW to accomplish the scoped work. Break it into executable
steps. Identify tools, approaches, and potential risks.

## Manager Role
- Read the Scope phase output for deliverables and acceptance criteria
- Design a step-by-step implementation approach
- Identify tools and resources needed per step
- Anticipate risks and prepare fallback approaches
- Order steps by dependency (what must happen first)

## Worker Instructions Template
Given the scope and acceptance criteria:
1. What is the step-by-step approach to implement this?
2. What tools/languages/frameworks are needed?
3. What order should steps be executed in?
4. What are the risks and how do we mitigate them?
5. How will we verify each step's output?

## Success Criteria Examples
- "Plan has at least N steps matching the scope"
- "Each step has clear inputs and outputs"
- "Dependencies between steps are identified"
- "Risk mitigation approaches documented"

## Common Pitfalls
- Over-planning simple tasks (a one-file change doesn't need 10 steps)
- Under-planning complex tasks (missing steps that become blockers)
- Not reading the scope's acceptance criteria (plan doesn't cover them)
