# Verify Phase Reference

## Purpose
Check whether the work ACTUALLY meets the acceptance criteria defined
in the Scope phase. Compare reality to expectations.

## Manager Role
- Read the Scope phase acceptance criteria from the task file
- Design verification steps that test each criterion
- Workers run tests, check outputs, validate behavior
- Compare Execute phase results to Scope acceptance criteria
- Score memories used across all phases for accuracy
- Score rules loaded across all phases for effectiveness

## Worker Instructions Template
Given the acceptance criteria and the Execute phase output:
1. For each acceptance criterion, verify it is met
2. Run any specified tests or checks
3. Report pass/fail for each criterion
4. Note any partial passes or edge cases

## Success Criteria Examples
- "All N acceptance criteria evaluated"
- "Each criterion has a clear pass/fail result"
- "Evidence provided for each evaluation"

## Common Pitfalls
- Rubber-stamping: saying "looks good" without actually checking
- Testing only the happy path
- Not reading the original Scope criteria (verifying against wrong thing)
- Being too lenient or too strict compared to actual criteria
